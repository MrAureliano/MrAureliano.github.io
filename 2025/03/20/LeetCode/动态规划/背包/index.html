3180. 执行操作可获得的最大总奖励 I

3181. 执行操作可获得的最大总奖励 II

定义f[i][j]表示能否从rewards的前i个数中得到总奖励j

设rewards的第i个数为v，考虑v选或不选

  • 不选v，问题变成能否从前i-1个数中得到总奖励j，即f[i][j] = f[i-1][j]
  • 选v，问题变成能否从前i-1个数中得到总奖励j-v，即f[i][j]=f[i-1][j-v]，前提是j满足j$\geq$v且j-v$<$v，即v$\leq$j$<$2v。

选或不选满足其一即可，f[i][j] = f[i-1][j] || f[i-1][j-v]

初始值f[0][0]=true ;

答案为最大的满足f[n][j] = true 的 j。

j最大枚举到2m-1，其中m是数组的最大值。

本题数据范围更大，需要去掉第一个维度，并用bitset优化（也可以用bigint）

优化一

转移方程去掉第一个维度得：f[j] = f[j] || f[j-v]，其中，v$\leq$j$<$2v。

进一步地，把一维数组压缩成一个二进制数f，其中二进制从低到高第j位为1表示f[j]=true，为0表示f[0]=false， | | 为或运算(OR)。

如v = 3：

  • f[0] OR f[3] ;
  • f[1] OR f[4] ;
  • f[2] OR f[5] ;

这相当于取f的低v位，在左移v位，然后OR到f中，即f |= (f&((1<<v)-1)) << v，具体来说：

  • (1<<v) -1得到一个低v位全为1的二进制树
  • f & ( (1<<v)-1 )得到f的低v位
  • ( f &( (1<<v)-1 ) ) << v把f & ( (1<<v) - 1 )左移v位
  • 最后与f计算按位或，更新到f中，即f | = (f & ( (1<<v) -1) ) << v

初始值f = 1。

==答案为f的最高位，即f的二进制长度减一==。

小优化：对rewards的数字进行去重，因为无法选两个一样的数。
